HTML Notes

  1. For ordered lists, use the ol tag and li for each list item.
    • Shut off bullets with 
      style="list-style-type: none"
  2. To preserve formatting, use the 
    < pre > tag
  3. To create horizontal lines, use the < hr > tag
  4. To bold, use < strong > tag and to italicize, use < em > tag
  5. For superscript and subscript, use < sup > and < sub > tag
  6. For hyperlinks, use 
    a href="file.html"
  7. For images, use 
    < img src="abc.jpg" />
  8. Meta tags
    • Redirect to another page using 
      < meta http-equiv="refresh" content="5; URL=https://www.google.com/" />
    • "Name" attribute can equal description, author, keywords, viewport
    • "Content" attribute will be the content of whatever specified in the name
  9. Doctype declaration tells browser which HTML version
    !DOCTYPE html
  10. Create tables with < table > tag
    • < tr > for rows
    • < td > for table data (columns for each row)
    • < th > for header rows that are bold and centered
    • & n b s p ;    for empty cell
    col 1 col 2 col 3
    hihi hihi hihi
  11. Create forms with < form > tag
    • < input > tag for text, radio, checkbox, submit, reset. Attribute required = "1" to make input required
    • Name attribute of radio buttons must be the same to shut off others when select one
    • Method attribute is get if you want data sent in query string. Method attribute is post if you want that sent to file and hidden from user
    • Action attribute is where data is goign

Git Notes

  • Stage changes by doing 
    git add --all
  • Commit changes by doing 
    git commit -m "my message"
  • Push changes by doing 
    git push -u origin main
  • Clone a repo (for Github pages) by doing 
    git clone https://github.com/username/username.github.io
  • Learn how to use Jekyll later. More info on https://pages.github.com/ and https://docs.github.com/en/github/working-with-github-pages/setting-up-a-github-pages-site-with-jekyll
